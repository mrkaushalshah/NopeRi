# Phase 06.1 Plan: Outreach Pipeline Optimization Summary

This phase focuses on refactoring the Local Company Outreach Engine to achieve massive speedups (up to 85–90%) and reduce LLM API calling costs by 50% without dropping quality.

## Tasks Planned
1. **Smart Scraping, Queue Inversion, and Timeout Refactoring**: Invert the crawl queue to prioritize homepage dynamic links containing career/contact keywords over blind guesses. Introduce early stopping upon finding corporate email prefixes (`hr@`, `careers@`, etc.). Reduce page fetch timeouts to 5 seconds and maximum pages to 4.
2. **Unified AI Enrichment (Single LLM Call)**: Combine the classification prompt and the email drafting prompt into a single consolidated JSON prompt. This cuts API roundtrip times in half and improves copy alignment.
3. **Parallel Places Discovery & Multi-Threaded Batch Concurrency**: Parallelize the 5 initial Google Places keyword searches using a thread pool. Transition the background FastAPI pipeline from sequential company processing to high-concurrency multi-threaded waves using Python's `ThreadPoolExecutor`.
