**Gathered:** 2026-06-01

**Source:** Outreach Pipeline Optimization Proposal (outreach_optimization_options.md)

<domain>
## Phase Boundary

This phase delivers a major performance and latency upgrade to the local outreach pipeline by refactoring the synchronous, sequential execution flow into a highly optimized, concurrent, and smart pipeline. It implements:
1. **Smart Scraping & Early Stopping** in website crawling to avoid unnecessary requests and prevent blocking.
2. **Unified AI Enrichment** via a single combined LLM call, reducing OpenAI API round-trips and cost.
3. **Multi-Threaded Batch Concurrency** to parallelize place details fetching, email scraping, and AI enrichment.
4. **Parallel Places Discovery** to run initial keyword searches concurrently.

</domain>

<decisions>
## Implementation Decisions

### 1. Smart Scraping & Early Stopping
- **Early Stopping**: Stop crawling a company website immediately once a high-quality email is found. Promising prefixes: `hr@`, `careers@`, `jobs@`, `recruitment@`, `hello@`, `contact@`, `work@`, `join@`, `hiring@`, `people@`.
- **Queue Prioritization (Queue Inversion)**: Parse the homepage first. Identify and enqueue actual links containing keywords `career`, `contact`, `jobs`, `hiring`, `about`, `team`. Do not append the 15 standard blind-guess paths to the queue first; only append 2-3 common blind guesses (e.g. `/careers` and `/contact-us`) as a last resort if no promising links are found on the homepage.
- **Shorter Timeouts & Pages**: Set the HTTP socket connection timeout to `5` seconds (instead of 10s). Set the maximum crawled pages (`max_pages`) per website to `4` (instead of 8).

### 2. Unified AI Enrichment
- **Combined Prompt**: Combine the two sequential LLM completion calls in `local_outreach.py` (classification & email drafting) into a single, unified LLM prompt.
- **JSON Schema**: The prompt must return a JSON object with:
  ```json
  {
    "is_software_company": true,
    "fit_score": 85,
    "reasoning": "Fits the profile...",
    "company_type": "product",
    "email_draft": {
      "subject": "Senior Full-Stack Engineer...",
      "body": "Dear HR Team..."
    }
  }
  ```
  If `is_software_company` is false OR `fit_score` is less than 40, `email_draft` should be `null`.

### 3. Multi-Threaded Batch Concurrency
- **Async Thread Pool in Background Task**: In `src/api/app.py`'s `background_pipeline`, replace the synchronous loop over companies with a `ThreadPoolExecutor` from `concurrent.futures`.
- **Worker Function**: Create `process_single_company(company, client, location, radius, db)` to handle place details, email scraping, unified AI enrichment, and DB persistence for a single company.
- **Batch Size Configuration**: Run the pool with `max_workers` configured based on the `mode` parameter:
  - `mode == "production"`: 5 parallel workers.
  - `mode == "development"`: 2 parallel workers.

### 4. Parallel Places Discovery
- **Parallel Searches**: In `search_companies` in `local_outreach.py`, run the queries in `SEARCH_QUERIES` in parallel using a thread pool.
- **Deduplication**: Merge and deduplicate the search results in-memory using `google_place_id` before proceeding.

</decisions>
