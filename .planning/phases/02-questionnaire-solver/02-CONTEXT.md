# Phase 2: Questionnaire Solver Implementation - Context

**Gathered**: 2026-05-03
**Status**: Ready for planning

## Phase Boundary
This phase implements the reasoning logic. It takes the "Professional DNA" from Phase 1 and applies it to specific job questionnaires encountered during the application process.

## Implementation Decisions

### AI Reasoning Logic
- **Solver**: Implement `solve_questionnaire(job_details, questions)` in `AIHandler`.
- **Context Injection**: Every solver call must include the extracted user profile + the Job Description.
- **Handling Options**: For multiple-choice questions, the AI must return the **index** of the correct option as expected by Naukri's API.

### Naukri Client Integration
- **Integration**: Update `src/client/job_client.py`.
- **Workflow**: 
  1. Try to apply.
  2. If questionnaire returned, call AI Solver.
  3. Submit the answers generated by AI.
  4. If successful, return the application result.

### the agent's Discretion
- Prompt engineering for the solver (making the AI act like a high-end recruiter/candidate).
- Handling "Self-Identification" or "Diversity" questions (usually skipping or choosing neutral options).

## Canonical References
- `src/client/job_client.py` — The core application logic.
- `01-ai-service/01-AI-SPEC.md` — The AI framework and model choice.

## Specific Ideas
- Log the AI's "Reasoning" to a debug file so the user can see why it chose certain answers.
- Implement a "Safety Check": If AI is unsure, log it and skip the job rather than giving a bad answer.
